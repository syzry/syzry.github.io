7道题做了5道。不知道为什么functional 这道题会被定成 easy 难度，虽然我确实有点太钻牛角尖了想着一定要逆出来搞得自己头晕脑胀的，唉……算了。

SIGNAL

没有混淆没有壳，逻辑就是按顺序执行一些函数，函数里直接对flag上各位置的值进行比较判断是否正确，函数逻辑差不多长这样

直接一个个把各位置比较的值抄下来就得到 flag 了

License

程序逻辑是对license的两部分进行不同的操作并比较字符串，判断是否正确。都是可逆的操作所以直接逆出来就可以了。但是输入的不是可见字符，所以调试的过程中花了一些时间。然后要连接远程服务器来获取flag。

BRAVE TRAVELER

一个小思路题。

题里算是建了个树，然后根是’\0’，’\0’连着’g’，程序就这样从’\0’开始一直 bfs 下去直到没有办法遍历。然后原始的程序直接就会输出 go0d luck.

后面自己的思路乱了，把这个树写出来了之后想着怎么输入，但后面回看题发现flag只需要输出的内容。

稍微分析一下就发现 flag 应该是从’{‘开始遍历，获得方法就是把’\0’连着的’g’改成’{‘就行了

VIRTUAL-REV

老老实实的 vm，就是输入字符串，调用对应的函数。有11个二元函数，6个一元函数，每个函数最多用10次。还有8个寄存器，在re里只用得到7个寄存器说实话，提示里说 Pwn 还要用到这个题，那剩下一个寄存器应该是给 pwn 用的。最后对于5个寄存器里的值进行判断，构造使用函数的方式就可以了。

对函数整理后有

STF 2 a1<<a2

x8

有点神经病的题，做的时候看到那么多人出了感觉自己像个sb做不出来，后面做出来了感觉还好，赛后看他们的做法又觉得自己像个sb。

rust 题。然后，简单来说，x8 程序本身是一个虚拟机，然后给了个 program.bin，里面有指令和数据。

本来打算理清程序逻辑，但 x8 是个 rust，搜出来也没 x8 的库（具体来说，没有搜到虚拟机作用的库），看得脑壳疼，本来打算放弃了，但是我搞不懂为什么这道题比virtual-rev那道题过的人都还多，所以继续看，发现有

在动调的时候，对第二个 readstdinstack 函数下断点，发现还没输入就停止，输入后跳转到函数外，搜索输入的字符串，发现就有了。所以这就是输入的函数。

后面的就是大胆假设，因为里面的看起来有用的函数就这些，实际上除了 cmp, xor 外还有 add，但在动调后发现 add 并不会对输入的值有什么影响（大概）。所以假设只进行了异或操作然后比较。

在第二个 xor 函数下断点后，发现

04是[rsi+rdi+401h]，04左边那个byte是[rsi+rax+401h]

在后面 cmp 函数下断点发现刚好就是04左边的 byte 和04右边的 byte 进行比较。但是我没有截图。

后面就是把所有运行到 xor 函数的断点时的04位置和66位置的值记下来，然后异或就得到 flag。

functional

赛中没做出来。这是道 haskell 的题，以前其实没做过 haskell 的题（），然后赛中做的时候净想着要逆出来，但是调试的时候很恼火，总是有很多 sigalarm 的异常，最后也没逆出来。

赛中用 hsdecomp 反编译了下，用的是 https://github.com/jespiron/hsdecomp 的版本。只知道长度要为28，以及最后要和28个大数比较。可能和模意义下的运算有关。

反编译结果：

赛后一看，全是侧信道攻击……各种八仙过海的侧信道攻击……但是也怪不得动调的时候那么多 sigalarm 的异常了，记下来这点了。每正确一个字符就要多运行明显一串时间。
